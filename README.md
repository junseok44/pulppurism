## 프로젝트 개요

### 문제 상황

- 지방 소멸 막기 위해서는 주민의 삶에 밀착된 정책이 필요함
- 정책을 지속적으로 제안, 논의할 지역 차원의 공론장 부재
- 주민 개인이 공식적인 정책 건의를 진행하기에는 심리적·행정적 부담 있음
- 마을 내 소통은 활발하나, 주민 의견이 객관적 데이터로 축적되지 않음
- 행정을 설득할 데이터와 유사 정책 사례 탐색, 관리의 어려움
- 반복되는 주민 수요가 정책으로 이어지지 못하고 단발성 의견에 그치는 구조 고착화

### 해결 아이디어

- 음성, 텍스트 기반의 간편한 주민 의견 입력 기능 제공
- AI를 활용한 유사 의견 자동 클러스터링
- 클러스터링된 의견을 하나의 정책 ‘안건’으로 정리
- 안건 단위로 주민 공감도(좋아요·댓글·투표), 참고 자료, 타 지역 사례 모음
- 정책 제안에 필요한 객관적 근거 데이터 확보
- 의견 → 안건 → 정책 제안 → 실행의 전 과정을 타임라인로 제공
- 정책 진행 상황 및 실행 결과에 대한 지속적인 피드백 제공

### 기대 효과

- 지역 문제를 효율적으로 해결한다.
- 지역 공동체 내 신뢰와 협력이 강화된다.
- 보도 및 행정 전반에서 부담이 경감된다.
- 정책 수립과 보도의 효율성과 정확성이 강화된다.

### 핵심 가치

- 접근성 : 누구나 가볍게 의견을 남길 수 있다.
- 축적성 : 주민 의견이 휘발되지 않고 기록되어 남는다.
- 연결성 : 개인의 의견을 공동의 안건과 정책으로 연결된다.
- 투명성 : 정책 제안과 실행 과정을 주민에게 지속적으로 공유한다.

## 코드 설치 및 실행 방법

### 필수 요구사항
- Node.js 20 이상
- PostgreSQL 데이터베이스 (또는 Neon 계정)
- npm 또는 yarn

### 설치

```bash
# 의존성 설치
npm install
```

### 환경 변수 설정

프로젝트 루트에 `.env` 파일을 생성하고 다음 변수들을 설정하세요:

```env
# 데이터베이스
DATABASE_URL=your_postgresql_connection_string

# 세션
SESSION_SECRET=your_session_secret

# 포트
PORT=5000

# OpenAI
OPENAI_API_KEY=your_openai_api_key

# OAuth (Google, Kakao)
GOOGLE_CLIENT_ID=your_google_client_id
GOOGLE_CLIENT_SECRET=your_google_client_secret

KAKAO_CLIENT_ID=your_kakao_client_id
KAKAO_CLIENT_SECRET=your_kakao_client_secret

BASE_URL=http://localhost:5000
```

### 실행

```bash
# 개발 모드 실행
npm run dev

# 프로덕션 빌드
npm run build

# 프로덕션 실행
npm start
```

### Docker를 통한 실행

#### Docker 이미지 빌드

```bash
# Docker 이미지 빌드
docker build -t pulppurism .
```

#### Docker 컨테이너 실행

```bash
# 환경 변수 파일(.env)을 사용하여 컨테이너 실행
docker run -d \
  --name pulppurism \
  -p 5000:5000 \
  --env-file .env \
  pulppurism
```
